Context-Aware Research Chatbot

Jan - Mar 2025 • Featured Case Study

The Challenge

Researchers were spending 3-5 hours daily searching for information across scattered sources—academic papers, web articles, technical documentation, and mathematical computations. They needed a unified intelligent system that could understand context, route queries appropriately, and deliver accurate answers instantly.

The Solution

Built an intelligent RAG-powered conversational AI system with multi-modal prompt routing that automatically classifies queries and selects the optimal tool—web search for current events, RAG for document-based queries, or mathematical engines for calculations.

  • Implemented FAISS vector store with semantic search for 10,000+ document retrieval
  • Engineered intelligent query router using LangChain with 92% classification accuracy
  • Built session-aware context management maintaining conversation history across interactions
  • Integrated Tavily for web search, Wolfram Alpha for math, and custom RAG pipeline

Technical Architecture

Stack: LangChain, FAISS, OpenAI GPT-4, Streamlit, Python
Key Components: Query classifier → Tool router → Response generator → Context manager

  • Similarity search with FAISS index optimized for sub-second retrieval
  • Prompt engineering with chain-of-thought reasoning for accurate routing
  • Streaming responses for better UX and lower perceived latency

Intelligent Incident Triage System

Nov 2024 - Jan 2025 • Case Study

The Challenge

Engineering teams were overwhelmed with 200+ incident reports weekly, spending 15-20 hours manually triaging, categorizing, and routing each issue. Human error led to 23% misclassification rate and critical incidents being missed or delayed by 4-6 hours on average.

The Solution

Developed a LangGraph-powered stateful workflow automation system that intelligently analyzes incident reports, classifies severity, extracts key evidence, and automatically routes to appropriate teams—with confidence scoring and human-in-the-loop fallback for complex cases.

  • Built conditional workflows with LangGraph supporting 7 decision nodes and 12 state transitions
  • Implemented confidence-based routing: >90% = auto-resolve, 70-90% = suggest action, <70% = human review
  • Trained on 5,000+ historical incidents achieving 85% severity classification accuracy
  • Integrated with GitHub Issues, Jira, and Slack for seamless workflow integration

Technical Architecture

Stack: LangGraph, GPT-4, FastAPI, PostgreSQL, Redis
Workflow: Intake → Evidence Gathering → Severity Analysis → Team Routing → Resolution Tracking

  • State persistence with Redis for handling long-running workflows across sessions
  • Retry logic with exponential backoff for API failures ensuring 99.7% uptime
  • Audit trail logging every decision for compliance and debugging
  • A/B testing framework comparing agent decisions vs human triage accuracy

Results & Impact

78% Full Automation
200+ Weekly Incidents
85% Classification Accuracy
16hrs Time Saved/Week
4min Avg Triage Time
7% Error Rate

Business Impact: Reduced triage time from 15-20 hours to 3-4 hours weekly, saving $12,000/month in engineering costs. Decreased incident response time from 6 hours to 4 minutes for automated cases. Misclassification rate dropped from 23% to 7%.

Lessons Learned

  • Start simple: V1 had 15 decision nodes—too complex. Simplified to 7 nodes improved reliability by 40%
  • Confidence thresholds are critical: Tuning from 80% to 90% reduced false positives by 60%
  • Human oversight builds trust: 22% human review rate gave teams confidence to adopt system
  • State management is hard: Debugging stateful workflows required custom visualization tools
