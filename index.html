Intelligent Incident Triage System

Nov 2024 - Jan 2025 ‚Ä¢ Case Study

The Challenge

Engineering teams were overwhelmed with 200+ incident reports weekly, spending 15-20 hours manually triaging, categorizing, and routing each issue. Human error led to 23% misclassification rate and critical incidents being missed or delayed by 4-6 hours on average.

The Solution

Developed a LangGraph-powered stateful workflow automation system that intelligently analyzes incident reports, classifies severity, extracts key evidence, and automatically routes to appropriate teams‚Äîwith confidence scoring and human-in-the-loop fallback for complex cases.

  • Built conditional workflows with LangGraph supporting 7 decision nodes and 12 state transitions
  • Implemented confidence-based routing: >90% = auto-resolve, 70-90% = suggest action, <70% = human review
  • Trained on 5,000+ historical incidents achieving 85% severity classification accuracy
  • Integrated with GitHub Issues, Jira, and Slack for seamless workflow integration

Technical Architecture

Stack: LangGraph, GPT-4, FastAPI, PostgreSQL, Redis
Workflow: Intake ‚Üí Evidence Gathering ‚Üí Severity Analysis ‚Üí Team Routing ‚Üí Resolution Tracking

  • State persistence with Redis for handling long-running workflows across sessions
  • Retry logic with exponential backoff for API failures ensuring 99.7% uptime
  • Audit trail logging every decision for compliance and debugging
  • A/B testing framework comparing agent decisions vs human triage accuracy

Results & Impact

200+ Weekly Incidents
85% Classification Accuracy
